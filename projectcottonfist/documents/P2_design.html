2. Client

Client name and contact person: Dropphase Ltd, represented by Cottonfist, Gunnar
Mr. Cottonfist is the CEO and Manager of Dropphase Ltd, and wishes to transition his business from a telephone service to an online service. Dropphase has been a local German company for many years, and now they wish to expand to the global market using Mr. Cottonfists many years of experience from the German puppeteering industry. Dropphase is a professional puppet rental service, providing puppets of any form and shape to any purpose necessary.

3. Purpose, goals, and audience

Mr. Cottonfist has achieved great success in his telephone services, but feels the growth has stagnated in recent years. Therefore, he wishes to expand to an internet service and take advantage of technological progress in order to make his business grow internationally.
Specifically, the goals he wants finished in the duration of this project is to have sections of puppets divided on how they function (sock puppets, hand puppets, animatronics, etc.), an about section where the clients may learn more about Mr. Cottonfist and his credibility, and an FAQ, where general questions regarding the company and/or details about the dolls.
The last goal he wants is to implement an ordering system, however in this project he only wants the visuals, and therefore not anything with the back-end yet.
The target audience will be enthusiasts, theaters, schools and students, and prop artists who need inspiration and reference on how puppets look like and work mechanically.

4. Navigation Structure

Our navigation section will consist of a main navigation bar located on the header area of the site and a basic sitemap in the footer section. Both the header and footer will be the same in all of the pages, connecting them like a web. This is to make it easier for the user to navigate from one part of the site to the other. Some of the site will also have linear structure, like from puppet info to browse puppet, to make the browsing experience seamless.

The main navigation bar will link to the following sites:
Puppet Info | Browse | Support | FAQ | About

On puppet info we will be adding a minimalistic drop-down menu when the user is hovering over the puppet info that links to the sub-puppet info pages to reduce navigation clicks (“NavClicks”).To navigate deeper into the website the user will use links located in the respective pages.Sitemap will have a direct link to the basic pages in our web site, nothing real exciting.

5. Page layout and Appearance

Since Dropphase ltd. is a relative unknown company (some might say imaginary) and their livelihood is depending on sales it is critical that the general design isn’t distracting and efficient. After Mr. Gunnars wishes are we keeping the universal design clean and professional as possible.
On the top of the page you will find a header that will consist of the company logo and the main navigation bar. As the user tends to favor the left (source) side we want the navigation bar to start with the sites that leads to revenue, in order Puppet info, Browse, Support, FAQ, About. The reasoning that we are putting Puppet info before Browse, is that if the user doesn't experience a drop-down button on the first nav bar, there is no expectation that there will be on the others, thus making an accidental “NavClick” if it isn’t expected.

Font-Type:

  • Logo: Coolvetica, this will be made as a .png/.svg file so the font won’t differ on the user experience
  • Body: Helvetica. We are using standard fonts on the body, to make the overall design available to most of the browsers, and avoid lack of fonts.

Color: The color theme has not yet decided upon as it is in disagreement between client and us. Since he wants neon yellow, cyan and pink color pallet, we are trying to get him to compromise, but at the current time we gave not been successful.

Social Media buttons: We are going to put the Social media button and the bottom right of the footer section, and in about, (NB! Since this is an imaginary company these buttons will link to the respective main sites.)

6. Content

Main page
The purpose of the main page is to introduce the user to Dropphase as a company and the services provided by them. The page will also inform the user about the different subsites with links to them.The very first time you visit the website you will get a splash page which will show the logo of Dropphase and the navbar covering the screen. By focusing on these two things it will be easier to navigate the website, and it will be clear what the website is all about. As the user scrolls down the splash screen will be condensed into the header and navbar for our website, and the user will see the main page. There the user will be introduced to the services of Dropphase, and get a more detailed look at the different subpages. By pressing any of the sections the user will be taken to the top and the corresponding part of the navbar expands.

Puppet info
The purpose of the puppet info page is to expand the users understanding of puppeteering and the different puppets provided by Dropphase. These different types will be split into boxes (html div elements) containing , and by pressing them the boxes will provide additional information and a link to the “browse puppets” subpage with the correlating filter for the puppet category. Example: If you click on the sock puppet section you will be redirected to the “browse puppets” subpage with the filter for sock puppets already checked out.

Browse puppets
The purpose of the browse puppet” page is to show the user the wide variety of different puppets provided by Dropphase. It also important that the user will be able to quickly find the right puppet for the right situation.
On the page where you browse the puppets available for rent the user will be able to filter different categories and search for the desired puppet. All this will be placed on the left side of the page while the focus will be on the right, where the puppets will be represented by an appropriate thumbnail, description and price. We will be using separate XML and JavaScript files to get information about the puppets.

About
The purpose of the about page is to give the user a complete history of Dropphase and Gunnar Cottonfist, with an honourable mention to the creators of the stupendous website that is being used.
The website will have bodies of text and accompanying pictures of Mr. Cottonfist.

Support page
The purpose of the support page is to give the user ways to contact Dropphase and thereby use their services or get necessary support. The website will have contact information for Dropphase with links to their social media. To make it clear which sites we are using, there will be logos acting as buttons.

FAQ
The purpose of the frequently asked questions page is to give the users answers to the most common questions. The page will focus on answering possible problems that the user might have, and giving insightful facts about puppeteering and Mr. Cottonfist.

7. Minimum Requirements

We currently have 5 JavaScripts we wish to implement to the site. The workload is therefore split up like this:
Mathias and Ninh will write and implement:

  1. A script which loops pulls information (thumbnail URL, name, description, and price) from an external XML file, and inserts it into the appropriate place. The reason we do this is because we have a plethora of inventory, and do not wish to create way too many html files that are almost identical. In addition, we do not have the ability to connect the site to a database and pull information from there. The script will function in the Browse Puppets section (browse.html), and will pull relevant info for each puppet.
  2. A script which handles search and filters for the first script. That way, the consumer is able to look at all of the puppets related to the search or the filter. The script will either refresh the entire page, or run actively and remove dynamically. It is also possible that this script is combined with the script above to reduce the workload of the page (avoid fetching everything and then filter out what’s not needed)
  3. A script that handles the transition from the splash screen to the Main Page (index.html). The script will condense the logo and shortcuts into the headers logo and navbar respectively. It is possible we will implement jQuery here, however, jQuery does not, as far as we know, condense the information, but only hides it

Nithun and Simen will write and implement:

  1. A script which expands DIVs. This is for both the Main Page (index.html) and Puppet Info (puppets.html), as they contain expandable boxes that contain info. The DIVs will expand to a certain width and height, revealing additional information, and a larger image. There will also be a close button to minimize the DIV
  2. A Scripts that stitches the header, content, and footer. This script will be on each page because every page needs it, however, due to the splash screen transition, it is possible this need to be “dormant” in a sense, until the splash screen disappears.
