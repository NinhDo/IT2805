Changes in scripts

stitch.js
Due to the inefficiency of stitching using javascript we decided using php with the include function instead

filter.js
We decided to let the filter dynamicly hide the irrelevant products instead of refreshing the entire page.

expand.js
Instead of a close button we made it so the box will collapse with a clock on the expanded box

General Structure changes
To increase the efficiency of the stitching we changed the pages to php instead of html. The front page, browse, and puppets have a special html-head to include the specific scripts they need. The other sites got a general html-head that doesn't include a unique title because we felt it was redundant to create a special html-head for each page.

The header includes only the header element and the opening element to the wrapper. Similarly the footer contains the footer element but also the closing element to the wrapper.

The content of each pages is kept in a separate html file to be used in the corresponding php file.

Everything that is not a php file was either kept in the asset, images, or scripts folders, with the exception of style.css.

Images used by the fetch script were kept in a separate xmlImages folder inside the images folder.

sytlesheet.css was changed to style.css
